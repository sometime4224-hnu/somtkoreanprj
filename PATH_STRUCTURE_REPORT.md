# Path Structure Report

## 목적

이 문서는 현재 저장소의 폴더 구조를 빠르게 이해하고, 무엇이 Git에 남아야 하는지와 무엇이 로컬 아카이브로 빠져야 하는지를 구분하기 위한 기준 문서이다.

## 현재 구조 요약

### Git에 남아야 하는 핵심 구조

- 루트 진입점: `index.html`
- 단원별 페이지: `c10/`, `c11/`, `c12/`, `c13/`, `c14/`
- 공용 자산: `assets/`, `shared/`
- 보조 앱과 독립 페이지: `apps/`
- 리뷰/평가 페이지: `review/`
- 개발 보조 코드: `scripts/`, `tests/`, `package.json`, `playwright.config.js`
- 핵심 문서: `README.md`, `LICENSE.md`, 구조/자산 가이드 문서

### Git에 오래 남기지 않는 편이 좋은 로컬 전용 항목

- 임시 작업물: `.codex_tmp/`, `tmp/`, `output/`
- 스크린샷, PDF 추출물, 비교 이미지, 일회성 실험 산출물
- 보관용 ZIP, 수동 백업 폴더, deferred deletion 묶음
- 개인 환경 설정: `.claude/`, `.git/info/exclude`

## 현재 상태 평가

현재 저장소는 배포 가능한 페이지와 코어 자산이 루트에서 비교적 명확하게 구분되는 상태다. 특히 `main` 브랜치 동기화, 브랜치 정리, 퀴즈 데이터 로컬라이즈 작업이 끝나면서 이전보다 훨씬 안정적인 구조가 되었다.

다만 아래 항목은 계속 남아 있으면 다시 혼선을 만들 가능성이 있다.

1. 로컬 임시 폴더가 저장소 안에 오래 남아 있으면 실제 소스와 스크래치 산출물의 경계가 흐려진다.
2. 로컬 전용 ignore 규칙은 다른 환경에서 재현되지 않기 때문에 추적 여부를 헷갈리게 만든다.
3. 장기 보관 목적의 백업 자료는 저장소 안보다는 프로젝트 바깥 아카이브 폴더가 더 적절하다.

## 권장 운영 규칙

1. 저장소 안에는 배포 대상, 소스, 공용 자산, 장기 유지 문서만 남긴다.
2. 스크린샷과 추출물은 저장소 안 임시 폴더에 오래 두지 말고, 작업이 끝나면 프로젝트 바깥 아카이브로 이동한다.
3. 공용 ignore는 `.gitignore`에 두고, `.git/info/exclude`는 가능하면 비워 둔다.
4. 실험적 기능은 로컬 임시 파일 누적보다 Git 브랜치로 격리하는 편이 안전하다.

- 로컬 임시 자료와 백업 자료는 저장소 바깥 아카이브로 이동하는 방향으로 운영 기준을 확정했다.
