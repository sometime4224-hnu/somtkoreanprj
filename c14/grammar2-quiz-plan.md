# 14과 문법 2 연습용 퀴즈 기획안

## 1. 기획 목적

- 대상 문법: `A/V-았던/었던`
- 핵심 목표: 과거의 경험이나 기억 속 대상을 떠올리며 명사를 꾸미는 표현을 퀴즈형 활동으로 반복 연습한다.
- 현재 `c14/grammar2.html` 안에 있는 기본 연습 6문항을 확장해, 별도 보조 퀴즈 1개로 학습 몰입도와 반복량을 늘린다.
- `문법 설명 -> 짧은 연습 -> 보조 퀴즈 -> 말하기 확장` 흐름을 완성하는 역할을 맡는다.

## 2. 권장 퀴즈 콘셉트

### 제목 제안

- 1안: `추억 앨범 퀴즈`
- 2안: `기억 속 장면 찾기`
- 3안: `그때 그 사람, 그곳`

### 추천 방향

`추억 앨범 퀴즈`를 추천한다.  
이 문법은 `사진`, `사람`, `곳`, `옷`, `친구`처럼 기억 속 명사를 꾸밀 때 가장 자연스럽기 때문에, 오래된 사진이나 메모를 다시 정리하는 콘셉트가 문법 의미와 잘 맞는다.

## 3. 학습 목표

- `-았던/었던`이 현재 진행이 아니라 과거의 경험, 기억, 완료된 장면을 떠올릴 때 쓰인다는 점을 익힌다.
- `만났던 사람`, `갔던 곳`, `찍었던 사진`, `입고 왔던 스웨터`, `친했던 친구` 같은 고빈도 묶음을 빠르게 인식한다.
- `-고 있는`, 현재형 관형사형, 미래 추측형 같은 비목표 표현과 구별한다.
- 짧은 문장 완성까지 스스로 할 수 있게 만든다.

## 4. 퀴즈 포맷 제안

- 형식: 모바일 우선 단일 HTML 퀴즈
- 권장 파일명: `c14/grammar2-memory-quiz.html`
- 난이도: `쉬움 -> 보통 -> 짧은 산출`
- 총 문항 수: 12문항
- 예상 플레이 시간: 4~6분
- 언어 구성: 한국어 기본, 베트남어 힌트/해설 선택 노출

## 5. 라운드 구성

### 라운드 1. 장면 고르기

- 문항 수: 4
- 형식: 4지선다
- 목표: `-았던/었던`이 들어간 문장을 보고 가장 자연스러운 장면 또는 문장을 고르게 한다.
- 포인트: 문법 의미를 직관적으로 연결

예시 주제

- 가족들과 여행 갔던 바다
- 어렸을 때 친했던 친구
- 비행기에서 만났던 사람
- 졸업 여행 때 찍었던 사진

### 라운드 2. 빈칸 복원

- 문항 수: 4
- 형식: 4지선다
- 목표: 명사 앞 관형형 자리에 `-았던/었던`을 넣는 감각을 강화한다.
- 포인트: 오답을 통해 현재 진행형, 희망 표현, 불필요한 연결형과 구별

오답 예시

- `찍고 있는`
- `입고 있는`
- `만나는`
- `갈수록`

### 라운드 3. 추억 문장 완성

- 문항 수: 4
- 형식: 짧은 서술형
- 목표: 제시어를 보고 한 문장을 완성한다.
- 포인트: 완전 자유 서술보다는 정답 패턴이 분명한 문장 완성형으로 설계

제시 방식 예시

- `어렸을 때 + 친하다 + 친구`
- `비행기에서 + 만나다 + 사람`
- `아침에 + 입고 오다 + 스웨터`
- `처음 + 만나다 + 곳`

## 6. 문항 설계 원칙

### 목표 표현

- `만났던`
- `친했던`
- `갔던`
- `찍었던`
- `입고 왔던`
- `살았던`

### 오답 설계 원칙

- 현재 진행과의 혼동 유도: `-고 있는`
- 단순 현재와의 혼동 유도: `-는`
- 미래/의지와의 혼동 유도: `-ㄹ`, `-고 싶어하는`
- 의미상 맞지만 문맥상 덜 자연스러운 표현도 일부 배치

### 어휘 범위

- 기존 `c14/grammar2.html` 예문과 같은 어휘를 우선 사용
- `사람`, `곳`, `친구`, `사진`, `스웨터`, `바다`, `여행`, `비행기`
- 목표 문법 외 다른 새 문법은 최소화

## 7. 화면 흐름 제안

1. 인트로 카드
2. 퀴즈 시작 버튼
3. 진행 표시 바
4. 문제 카드
5. 정답/오답 즉시 피드백
6. 최종 결과 카드
7. `grammar2.html` 복습 또는 `grammar1-2-speaking.html` 이동 버튼

### 결과 문구 예시

- 10~12개 정답: `추억 표현을 아주 자연스럽게 잡았어요. 이제 말하기로 이어 가도 좋습니다.`
- 7~9개 정답: `핵심은 잘 이해했어요. 헷갈린 표현만 한 번 더 보면 금방 안정됩니다.`
- 0~6개 정답: `현재 장면과 추억 장면을 구별하는 연습을 한 번 더 해 보세요.`

## 8. 샘플 문항

### 샘플 1

- 유형: 객관식
- 문제: `“졸업 여행 갔을 때 ____ 사진을 보니까 옛날 생각이 많이 나요.”`
- 선택지: `찍었던 / 찍고 있는 / 찍어하는 / 찍을수록`
- 정답: `찍었던`
- 해설 포인트: 과거에 찍은 사진을 떠올리며 명사를 꾸민다.

### 샘플 2

- 유형: 객관식
- 문제: `추억이 담긴 장소를 설명할 때 가장 자연스러운 문장을 고르세요.`
- 선택지:
  - `여기는 제가 작년 겨울에 가족들과 여행 갔던 바다입니다.`
  - `여기는 제가 여행 가고 있는 바다입니다.`
  - `여기는 제가 여행 갈수록 바다입니다.`
  - `여기는 제가 여행 가고 싶어하는 바다입니다.`
- 정답: `여기는 제가 작년 겨울에 가족들과 여행 갔던 바다입니다.`

### 샘플 3

- 유형: 서술형
- 문제: `“어렸을 때 + 친하다 + 친구”를 사용해 문장을 완성하세요.`
- 정답 예시: `어렸을 때 친했던 친구와 다시 연락하게 됐어요.`
- 채점 방식: 공백 제거 후 완전일치, 필요하면 허용답안 1~2개 추가

### 샘플 4

- 유형: 서술형
- 문제: `“비행기에서 + 만나다 + 사람”을 사용해 문장을 만드세요.`
- 정답 예시: `한국에 올 때 비행기에서 만났던 사람과 아직도 연락하고 있어요.`
- 해설 포인트: 과거 경험 속 사람을 현재 시점에서 설명하는 구조

## 9. 데이터 구조 제안

```js
{
  id: "g2m_01",
  type: "mcq", // mcq | short
  grammar: "A/V-았던/었던",
  scene: "졸업 여행 사진",
  q: "“졸업 여행 갔을 때 ____ 사진을 보니까 옛날 생각이 많이 나요.”",
  hintKo: "사진은 과거에 찍은 것입니다.",
  hintVi: "Bức ảnh là thứ đã chụp trong quá khứ.",
  options: ["찍었던", "찍고 있는", "찍어하는", "찍을수록"],
  answer: "찍었던",
  answers: ["어렸을 때 친했던 친구와 다시 연락하게 됐어요."],
  explainKo: "과거에 찍은 사진을 떠올릴 때 -었던을 쓰는 것이 자연스럽습니다.",
  explainVi: "Dùng -었던 là tự nhiên khi nói về bức ảnh đã chụp trước đây."
}
```

## 10. 구현 메모

- UI 스타일은 `c14/grammar1-mini-game.html`과 `c13/grammar2-mini-game.html`의 중간 톤이 잘 맞는다.
- 별도 이미지 없이도 시작 가능하지만, 콘셉트를 살리려면 `폴라로이드 사진 카드` 느낌의 일러스트 4~6장이 있으면 좋다.
- 기존 페이지와 맞추기 위해 아래 요소를 유지한다.
- 상단 글로벌 네비게이션
- 한국어/베트남어 토글
- 즉시 피드백
- 재도전 버튼
- 챕터 허브 복귀 링크

## 11. 허브 연결 제안

`c14/index.html`의 문법 2 카드 아래에 보조 퀴즈 링크 1개를 추가하면 흐름이 자연스럽다.

- lesson kind: `보조 퀴즈`
- lesson title: `추억 앨범 퀴즈`
- lesson note: `과거 경험과 기억 속 명사를 -았던/었던으로 빠르게 구별하며 연습하기`

## 12. 우선순위

1. 12문항짜리 퀴즈 페이지 제작
2. 허브 링크 연결
3. 필요하면 이미지 자산 추가
4. 이후 말하기 퀴즈와 결과 동선 연결

## 13. 한 줄 결론

`c14` 문법 2 보조 퀴즈는 `추억 앨범` 콘셉트의 12문항 모바일 퀴즈로 설계하는 것이 가장 자연스럽고, 현재 본문 예문과도 바로 연결된다.
