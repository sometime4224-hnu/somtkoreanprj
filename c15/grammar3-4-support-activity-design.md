# c15 문법 3·4 보조 활동 설계안

## 공통 방향

- 연습문제에 나온 상황과 어휘를 그대로 확장해서 낯선 활동처럼 보이지 않게 한다.
- 단순 객관식 반복이 아니라, 선택 후 직접 문장을 만들거나 말하게 한다.
- 학생이 "이건 진짜 생활에서 쓸 수 있겠다"라고 느끼는 장면을 우선한다.
- 화면은 설명보다 과제 중심으로 설계하고, 첫 화면에서 바로 활동이 시작되게 한다.

## 문법 3 보조 활동

### 추천 활동명

`꼭 해야 해! 생활 조건 미션`

### 핵심 문법

`A/V-지 않으면 안 되다`

### 연습문제 기반 핵심 소재

- 방을 구할 때 전기 요금 포함 여부 확인
- 1학년 신입생 수강 신청
- 분리수거 규칙
- 황사 때 마스크 착용
- 외출 후 손 씻기
- 밀린 요금 납부 후 전기 복구
- 중요한 면접 전 복장 준비

### 활동 콘셉트

학생이 여러 생활 상황에서 "하지 않으면 곤란한 일", "반드시 충족해야 하는 조건"을 빠르게 판단하고 말하는 미션형 페이지입니다.

기존 연습문제가 뜻 이해와 형태 선택 중심이었다면, 이 보조 활동은 "지금 이 상황에서 꼭 해야 하는 행동"을 바로 골라 문장으로 완성하게 합니다.

### 추천 화면 구조

1. 상황 카드
2. 해야 하는 행동 선택 카드 3~4개
3. 문장 완성 바
4. 결과 피드백
5. 다음 미션

### 한 문제의 진행 방식

1. 상황이 짧게 제시된다.
   예: "전기가 끊겼어요. 다시 들어오게 하려면?"
2. 학생은 여러 행동 카드 중 `필수 조건` 하나를 고른다.
3. 아래 문장 바에서 `밀린 요금을 빨리 내지 않으면 안 돼요`처럼 완성한다.
4. 맞으면 "해결됨", 틀리면 "아직 안 됨"처럼 결과가 즉시 보인다.
5. 가능하면 마지막에 한 번 말하기 버튼으로 직접 말하게 한다.

### 흥미 요소

- `문제 해결률` 게이지
- 선택 즉시 뜨는 `해결됨 / 아직 부족함` 결과 카드
- 생활 알림 UI
  예: 관리비 알림, 학교 공지, 생활 수칙, 친구 메시지
- 후반부에는 "규칙"이 아니라 "해결 조건"과 "준비 조건"을 섞어서 난도를 올린다.

### 이 활동이 실용적인 이유

- 문법의 핵심인 `필수 조건` 감각을 실제 생활 판단과 연결한다.
- 주거, 학교, 건강, 행정, 면접처럼 바로 쓰일 수 있는 장면을 다룬다.
- "해야 해요"와 "하지 않으면 안 돼요"의 말맛 차이도 자연스럽게 체감하게 한다.

### 추천 문제 세트

- 방 계약 전에 무엇을 꼭 확인해야 하나?
- 전기를 다시 쓰려면 무엇을 먼저 해야 하나?
- 면접이 있을 때 무엇을 준비하지 않으면 안 되나?
- 신입생은 수강 신청에서 무엇을 하지 않으면 안 되나?
- 황사가 심한 날에는 무엇을 하지 않으면 안 되나?
- 외출 후에는 무엇을 하지 않으면 안 되나?
- 재활용 쓰레기는 어떻게 버리지 않으면 안 되나?
- 생활비가 부족할 때 무엇을 하지 않으면 안 되나?

### 구현 추천 메모

- 추천 파일명: `c15/grammar3-must-do-mission.html`
- 시각 톤: 공지, 경고, 해결 배지 중심
- 입력 방식: 선택 + 문장 조립 + 짧은 말하기
- 결과 문구는 길게 설명하지 말고 `필수 조건 맞음`, `이건 그냥 선택 사항`, `아직 해결 조건이 아님`처럼 짧게 처리

## 문법 4 보조 활동

### 추천 활동명

`가는 길에 부탁 메신저`

### 핵심 문법

`V-는 길에`

### 연습문제 기반 핵심 소재

- 학교에 오는 길에 케이크 사기
- 부산에 가는 길에 경주 들르기
- 귀국하는 길에 면세점 부탁하기
- 운동 갔다 오는 길에 찾으러 가기
- 나가는 길에 음식물 쓰레기 버리기
- 퇴근하는 길에 우유 사기
- 들어오는 길에 택배 찾기

### 활동 콘셉트

학생이 상대의 이동 동선을 보고, 그 길에 자연스럽게 붙일 수 있는 부탁이나 계획을 골라 메신저 답장처럼 보내는 활동입니다.

기존 `grammar4-route-animation.html`이 문법 이해용이라면, 이 보조 활동은 실제 대화 적용용입니다.

### 추천 화면 구조

1. 현재 동선 카드
2. 부탁 또는 계획 카드 목록
3. 메신저 답장 입력 영역
4. 자연스러움 피드백
5. 다음 상황

### 한 문제의 진행 방식

1. 상단에 현재 동선이 제시된다.
   예: "친구: 나 지금 집에 들어가."
2. 아래에 가능한 부탁 카드가 나온다.
   예: `택배 찾아오기`, `내일 시험 공부하기`, `손 씻기`
3. 학생은 동선에 맞는 행동을 고른다.
4. `그럼 들어오는 길에 택배 좀 찾아 줄래요?`처럼 문장을 완성한다.
5. 맞으면 `동선에 잘 맞아요`, 틀리면 `이건 이동 경로가 안 느껴져요`처럼 짧게 보여 준다.

### 흥미 요소

- 메신저 말풍선 UI
- `동선 적합도` 표시
- 카드 드래그 또는 탭 조합
- 부탁하기 / 계획 말하기 / 설명하기를 섞어서 반복감 줄이기
- 실제 이동 경로가 아닌 보기에는 `이동 경로 아님` 효과를 줘서 문법 4의 핵심을 분명히 한다.

### 이 활동이 실용적인 이유

- 학생들이 일상에서 가장 자주 쓰는 장면인 부탁, 심부름, 일정 설명과 바로 연결된다.
- `-는 길에`를 아무 동사에 붙이면 안 된다는 점을 실제 동선 판단으로 익히게 한다.
- 친구, 가족, 직장, 학교 상황 모두에 응용할 수 있다.

### 추천 문제 세트

- 친구가 나가는 중일 때 음식물 쓰레기 부탁하기
- 친구가 집에 들어오는 중일 때 택배 부탁하기
- 퇴근 중일 때 우유 부탁하기
- 학교에 오는 길에 케이크를 산 상황 이해하기
- 부산에 가는 길에 경주를 들르는 계획 이해하기
- 귀국하는 길에 면세점 심부름 부탁하기
- 운동 갔다 오는 길에 물건 찾으러 가는 말 이해하기
- 실제 이동 동선이 있는 문장과 없는 문장 구별하기

### 구현 추천 메모

- 추천 파일명: `c15/grammar4-route-request-messenger.html`
- 기존 `grammar4-route-animation.html`과 역할 분리:
  이해 보조는 애니메이션, 적용 보조는 메신저 활동
- 입력 방식: 선택 + 문장 조립 + 짧은 말하기
- 피드백 문구는 `부탁으로 자연스러움`, `동선이 안 맞음`, `이건 규칙 문법에 가까움`처럼 매우 짧게 유지

## 우선 제작 순서

1. 문법 3 `꼭 해야 해! 생활 조건 미션`
2. 문법 4 `가는 길에 부탁 메신저`

문법 3은 아직 별도의 대표 보조 활동이 없어서 우선순위가 더 높습니다.
문법 4는 기존 도식 애니메이션이 이미 있으므로, 이번 설계안은 그 다음 단계의 적용형 활동으로 두는 것이 좋습니다.
