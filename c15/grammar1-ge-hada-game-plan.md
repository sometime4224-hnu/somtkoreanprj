# 15과 문법 1 `-게 하다` 게임 상세 기획안

## 1. 기획 검토 요약

현재 기본 아이디어는 아주 좋습니다.

- 학생이 `내가 직접 한다`와 `다른 사람이 하도록 만든다`를 한 화면에서 비교할 수 있습니다.
- `드래그 앤 드롭` 방식이어서 초급 학습자도 부담 없이 참여할 수 있습니다.
- 성공 시 애니메이션을 보여 주면 문법 의미가 장면으로 남아서 설명형 문제보다 기억에 오래 남습니다.

다만 실제 수업용 게임으로 만들 때는 아래 3가지를 더 분명하게 잡는 것이 좋습니다.

1. 학생이 무엇을 판단해야 하는지 선명해야 합니다.  
   단순히 "누구에게 드롭할까"만 맞히는 게임이 되면 문법 연습보다 캐릭터 맞히기 게임이 될 수 있습니다.

2. `동사 + 명사` 조합 오류를 시스템이 잘 처리해야 합니다.  
   예를 들어 `영화를 먹다`, `책을 마시다` 같은 조합은 자연스럽게 배제하거나 오답 피드백으로 처리해야 합니다.

3. 성공 장면에서 반드시 `-게 하다` 문장을 다시 들려줘야 합니다.  
   학생이 행동만 보고 끝내면 문법 형식을 놓칠 수 있으므로, 결과 화면에서 문장을 다시 보여 주는 장치가 필요합니다.

## 2. 게임 목표

- 학습 목표 1: `나`가 직접 하는 행동과 `남이 하도록 시키는 행동`을 구분한다.
- 학습 목표 2: `A가 V하다`와 `A가 B를/에게 V-게 하다`의 의미 차이를 장면으로 이해한다.
- 학습 목표 3: 자주 쓰는 동사와 목적어를 연결하며 `-게 하다` 문장을 반복 노출한다.

## 3. 추천 게임 콘셉트

### 제목 후보

- `하게 하다 미션`
- `누가 하게 할까요?`
- `시켜라! 행동 작전`
- `내가 할까, 하게 할까?`

### 추천 콘셉트 문장

`지시문을 읽고 알맞은 행동을 완성하세요. 내가 직접 할지, 다른 사람이 하게 할지 잘 판단해야 합니다.`

## 4. 핵심 게임 구조

한 문제는 아래 순서로 진행하는 것을 권장합니다.

1. 상단에 미션 제시
2. 학생이 `행동 카드`를 읽음
3. `명사 카드`와 `동사 카드`를 선택
4. 완성한 행동 묶음을 `나` 또는 `타인`에게 드래그
5. 정답이면 문장 + 애니메이션 + 음성 효과 출력
6. 오답이면 왜 틀렸는지 짧게 피드백

### 미션 유형은 2종으로 단순화

#### A형: 내가 직접 하는 행동

- 예시: `케이크를 먹으세요.`
- 정답 행동: `케이크 + 먹다`
- 정답 대상: `나`
- 결과 문장: `제가 케이크를 먹어요.`

#### B형: 타인이 하도록 만드는 행동

- 예시: `친구가 케이크를 먹게 하세요.`
- 정답 행동: `케이크 + 먹다`
- 정답 대상: `친구`
- 결과 문장: `제가 친구에게 "케이크를 먹어."라고 말해요.`
- 문법 강조 문장: `친구가 케이크를 먹게 해요.`

이렇게 하면 학생은 항상 아래 두 가지를 동시에 판단하게 됩니다.

- 행동이 무엇인지
- 행동 주체가 누구인지

## 5. 화면 구성안

## 상단 영역

- 진행도: `3 / 10`
- 점수 또는 별 개수
- 현재 문법 태그: `문법 1 | -게 하다`

## 미션 영역

- 큰 글씨로 지시문 제시
- 아래에 작은 힌트 문장 제공
- 예시:
  - `빵을 먹으세요.`
  - `동생이 빵을 먹게 하세요.`

## 무대 영역

- 왼쪽 캐릭터: `나`
- 오른쪽 캐릭터: `친구`, `동생`, `부하직원`, `학생` 등
- 각 캐릭터 아래에 드롭 존 표시
  - `내가 할게요`
  - `이 사람이 하게 할게요`

## 카드 영역

- 명사 카드 줄
- 동사 카드 줄
- 선택된 카드가 위쪽 `행동 슬롯`에 조합됨

예시:

- 명사 카드: `빵`, `밥`, `케이크`, `물`, `책`, `신문`, `영화`, `드라마`
- 동사 카드: `먹다`, `마시다`, `읽다`, `보다`, `웃다`, `울다`

## 피드백 영역

- 정답 여부
- 완성 문장 다시 제시
- 간단 해설
- 다음 문제 버튼

## 6. 상호작용 규칙

### 추천 방식: `2단계 선택 + 드래그`

완전 자유 드래그보다 아래 방식이 학습자에게 더 명확합니다.

1. 명사 카드 1개 선택
2. 동사 카드 1개 선택
3. 화면 중앙에 `행동 묶음` 생성
   - 예: `케이크를 먹다`
4. 그 행동 묶음을 `나` 또는 `타인`에게 드래그

이 방식이 좋은 이유는 다음과 같습니다.

- 학생이 현재 만든 행동을 눈으로 확인할 수 있습니다.
- 잘못된 조합을 만들었을 때 바로 수정할 수 있습니다.
- 모바일에서도 조작이 덜 복잡합니다.

### 오답 판정 기준

- 명사와 동사가 맞지 않음
- 행동 대상은 맞지만 주체가 틀림
- 주체는 맞지만 행동 조합이 틀림

### 추천 오답 피드백

- `이 행동은 제가 해야 해요.`
- `이 행동은 친구가 하게 해야 해요.`
- `책은 먹지 않아요. 다시 골라 보세요.`
- `행동은 맞아요. 그런데 누가 해야 하는지 다시 보세요.`

## 7. 콘텐츠 설계

### 1차 버전에서는 어휘를 너무 넓히지 않는 것이 좋음

처음부터 모든 동사와 명사를 자유 조합하게 하면 오답 종류가 너무 많아집니다.  
1차 버전은 `호환 가능한 조합`만 중심으로 구성하는 것이 좋습니다.

### 권장 조합표

| 동사 | 자연스러운 명사 |
|---|---|
| 먹다 | 빵, 밥, 케이크 |
| 마시다 | 물, 커피, 우유 |
| 읽다 | 책, 신문, 편지 |
| 보다 | 영화, 드라마, 만화 |
| 웃다 | 없음 또는 상황 카드 |
| 울다 | 없음 또는 상황 카드 |

### 자동사 처리 권장안

`웃다`, `울다`는 목적어가 없는 동사이므로 따로 분리하는 것이 좋습니다.

- 방법 A: 자동사 라운드를 별도로 만든다.
  - 예: `웃으세요.`
  - 예: `동생이 울게 하세요.`

- 방법 B: 명사 없이 `행동 카드` 한 장으로 낸다.
  - 예: `웃다`
  - 예: `울다`

1차 버전은 구현이 쉬운 방법 A를 권장합니다.

## 8. 문제 난이도 설계

### 라운드 1. 주체 구분 연습

- 목표: `나`와 `타인` 구분
- 사용 어휘: `먹다`, `마시다`
- 예시:
  - `물을 마시세요.`
  - `친구가 물을 마시게 하세요.`

### 라운드 2. 동사-명사 조합 연습

- 목표: 자연스러운 조합 찾기
- 사용 어휘: `먹다`, `마시다`, `읽다`, `보다`
- 예시:
  - `신문을 읽으세요.`
  - `동생이 영화를 보게 하세요.`

### 라운드 3. 인물 바꾸기

- 목표: `친구`, `동생`, `부하직원`, `학생` 등 다양한 타인 표현 익히기
- 예시:
  - `부하직원이 신문을 읽게 하세요.`
  - `학생이 책을 읽게 하세요.`

### 라운드 4. 자동사 추가

- 목표: 목적어 없는 동사까지 확장
- 예시:
  - `웃으세요.`
  - `동생이 울게 하세요.`

## 9. 애니메이션/연출 제안

### 정답 연출: 내가 직접 하는 경우

- `나` 캐릭터가 행동
- 말풍선: `제가 케이크를 먹어요.`
- 행동 이펙트: 먹는 모션, 책 읽는 모션 등

### 정답 연출: 타인이 하게 하는 경우

- `나` 캐릭터 말풍선 출력
  - `케이크를 먹어.`
- 이어서 `타인` 캐릭터가 행동
- 마지막에 문법 강조 배너
  - `친구가 케이크를 먹게 했어요.`

### 추천 포인트

- `지시 말풍선`과 `결과 문장`을 둘 다 보여 주면 의미 연결이 훨씬 선명합니다.
- 정답마다 같은 구조로 반복 노출하면 학생이 문형을 자연스럽게 익힙니다.

## 10. 학습 피드백 문안

### 정답일 때

- `맞았어요! 이번에는 제가 직접 해요.`
- `맞았어요! 이번에는 친구가 하게 했어요.`
- `좋아요. "친구가 케이크를 먹게 했어요."`

### 오답일 때

- `이번에는 제가 직접 해야 해요.`
- `이 명사와 동사는 잘 어울리지 않아요.`

### 라운드 종료 피드백

- `직접 하는 행동과 하게 하는 행동을 잘 구분했어요.`
- `이제 "친구가 먹게 하세요" 같은 문장을 더 빨리 읽을 수 있어요.`

## 11. 점수/동기 요소

### 추천 방식

- 정답 1개당 별 1개
- 연속 정답 3개면 보너스 반짝 효과

## 15. 최종 권장안

이번 게임은 아래 방향으로 잡으면 가장 안정적입니다.

1. `나가 직접 한다`와 `타인이 하게 된다`의 대비를 핵심 규칙으로 유지한다.
2. `명사 + 동사`를 먼저 조합한 뒤 캐릭터에게 드롭하는 구조로 단순화한다.
3. 정답 후에는 반드시 `지시 말풍선 + 결과 문장`을 함께 보여 준다.
4. 1차 버전은 목적어가 분명한 동사 위주로 만들고, 자동사는 후반 라운드로 분리한다.
5. 현재 프로젝트 구조상 별도 보조 게임 파일로 만드는 방향으로 진행한다.
